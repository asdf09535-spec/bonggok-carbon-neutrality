🔥 Firebase 연결 가이드

봉곡중학교 탄소중립 학급 홈페이지 · 단계별 설정 방법

STEP 1

🌐 Firebase 프로젝트 만들기

1. https://console.firebase.google.com 에 접속해 Google 계정으로 로그인하세요.

2. [ 프로젝트 추가 ] 버튼을 클릭합니다.

3. 프로젝트 이름을 입력합니다. (예: bonggok-carbon)

4. Google 애널리틱스는 사용 안 함을 선택하고 프로젝트를 생성합니다.

STEP 2

📱 웹 앱 추가 및 설정값 복사

1. 프로젝트 홈에서 </> (웹) 아이콘을 클릭합니다.

2. 앱 닉네임을 입력하고 (예: 탄소중립-웹) [ 앱 등록 ]을 클릭합니다.

3. 아래와 같은 firebaseConfig 코드가 표시됩니다. 전체를 복사해 두세요!

// 이런 형태의 코드가 나옵니다 (값은 본인 것으로 다름)
const firebaseConfig = {
  apiKey:            "AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXX",
  authDomain:        "bonggok-carbon.firebaseapp.com",
  projectId:         "bonggok-carbon",
  storageBucket:     "bonggok-carbon.appspot.com",
  messagingSenderId: "123456789012",
  appId:             "1:123456789012:web:abcdefabcdef"
};
⚠️ 주의: 이 값들은 외부에 공개해도 되지만, Firestore 보안 규칙을 반드시 설정해야 합니다 (STEP 4 참고)
STEP 3

📦 Firestore 데이터베이스 활성화

1. 왼쪽 메뉴에서 빌드 → Firestore Database를 클릭합니다.

2. [ 데이터베이스 만들기 ] 버튼을 클릭합니다.

3. 위치는 asia-northeast3 (서울) 을 선택합니다.

4. 보안 규칙은 일단 테스트 모드로 시작합니다. (나중에 STEP 4에서 수정)

5. [ 완료 ]를 클릭하면 데이터베이스가 생성됩니다.

STEP 4

🔒 Firestore 보안 규칙 설정

Firestore Database → [ 규칙 ] 탭을 클릭하고, 아래 내용으로 교체 후 게시하세요.

// 누구나 읽기 가능, 쓰기도 가능 (학교 내부용 설정)
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read: if true;
      allow write: if true;
    }
  }
}
💡 학교 내부 사용 목적이므로 위 규칙으로 충분합니다. 외부 공개 사이트라면 인증 규칙을 추가해야 합니다.
STEP 5

✏️ shared.js 파일에 설정값 붙여넣기

다운로드한 shared.js 파일을 텍스트 편집기(메모장, VS Code 등)로 열고, 상단의 firebaseConfig 부분을 찾아 STEP 2에서 복사한 값으로 교체합니다.

// shared.js 상단 - 아래 부분을 본인 값으로 교체
const firebaseConfig = {
  apiKey:            "← Firebase Console에서 복사한 값",
  authDomain:        "← Firebase Console에서 복사한 값",
  projectId:         "← Firebase Console에서 복사한 값",
  storageBucket:     "← Firebase Console에서 복사한 값",
  messagingSenderId: "← Firebase Console에서 복사한 값",
  appId:             "← Firebase Console에서 복사한 값"
};

비밀번호도 이 파일에서 변경할 수 있습니다:

const ADMIN_PW = "1234"; // ← 원하는 비밀번호로 변경
STEP 6

📁 GitHub에 파일 올리기

아래 7개 파일을 모두 GitHub 저장소 같은 폴더에 올리면 됩니다.

index.html pledge.html books.html market.html diary.html shared.css shared.js ← 설정값 수정 필수!

GitHub 저장소에서 Add file → Upload files로 한꺼번에 업로드하거나, 기존 파일을 클릭해서 내용을 붙여넣기하면 됩니다.

⚠️ shared.js를 반드시 먼저 수정한 뒤 업로드하세요! 설정값이 없으면 사이트가 작동하지 않습니다.
STEP 7

✅ 동작 확인

GitHub Pages 배포 후 사이트에 접속해서 확인하세요:

  • 서약서 / 도서소감 / 중고장터 / 일기 탭에서 항목을 직접 작성해 보기
  • Firebase Console → Firestore Database에서 데이터가 실시간으로 쌓이는지 확인
  • 다른 기기(휴대폰 등)에서 접속해도 같은 데이터가 보이는지 확인
  • 🔐 관리자 버튼 → 비밀번호 입력 → 수정/삭제 버튼 확인
🎉 여기까지 완료되면 Firebase 연결 완료! 이제 모든 학생이 실시간으로 데이터를 공유합니다.

🌍 연결 완료!

Firebase Firestore를 통해 모든 학생이 실시간으로 같은 데이터를 봅니다.
한 학생이 서약서를 올리면 → 다른 학생 화면에 즉시 반영됩니다! 🌿

문제가 생기면 브라우저 개발자도구(F12) → Console 탭에서 오류 메시지를 확인하세요.
